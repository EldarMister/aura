# Бильярд — POS

Лёгкое приложение для управления бильярдными столами на **React Native (Expo) + TypeScript**.
Дизайн повторяет референсы из папки [`дизайн/`](./дизайн): белый фон, тёмный текст,
зелёный акцент, тонкие линии.

Есть два режима связи с телевизором:

- **Облако** — `socket.io` к внешнему серверу, работает в Expo Go.
- **Локально (Wi-Fi)** — прямое TCP-соединение телефон ↔ ТВ без ПК и интернета.
  В этом режиме ТВ поднимает сервер, показывает свой IP на экране, а телефон подключается
  к этому IP и отправляет снимки столов. Для локального режима нужен dev/EAS-билд,
  потому что используется нативный модуль `react-native-tcp-socket`.

Для облачного режима URL сервера и код заведения можно не вводить в приложении,
а прописать в `.env`:

```bash
EXPO_PUBLIC_SYNC_URL=https://your-sync-server.example.com
EXPO_PUBLIC_SYNC_ROOM=club-1
```

Если `EXPO_PUBLIC_SYNC_URL` задан, экран настроек скрывает ручной ввод URL/кода
и использует значения из `.env`.

Для облачного хранения добавлен сервер в `server/`: он принимает live-события через
socket.io и сохраняет последнее состояние заведения в Postgres (`DATABASE_URL`).
В состояние входят столы, тарифы, напитки, история игр и счётчик номеров игр.

## Запуск

```bash
npm install
npm start          # затем нажмите a (Android), i (iOS) или отсканируйте QR в Expo Go
```

Expo Go подходит для локальной разработки UI и облачного режима. Для прямого Wi-Fi:

```bash
npm run eas:android:dev
```

Проверка типов:

```bash
npm run lint       # tsc --noEmit
```

## Cloud sync server

Локальная проверка сервера без БД:

```bash
npm run server:install
npm run server:start
```

Без `DATABASE_URL` сервер хранит данные только в памяти — это подходит для теста,
но не для продакшена.

Для продакшена задайте переменные окружения на хостинге:

```bash
PORT=3000
DATABASE_URL=postgresql://user:password@host:5432/database
```

Сервер сам создаёт таблицу `cloud_state` при старте. В приложении укажите URL
этого сервера в `EXPO_PUBLIC_SYNC_URL`, например `https://billiard-sync.example.com`.

## Экраны

| Экран | Файл | Что делает |
| --- | --- | --- |
| Управление столами | `src/screens/TablesScreen.tsx` | выбор стола, открытие игры, таймер, сумма, напитки, сетка действий 2×2 |
| Настройки | `src/screens/SettingsScreen.tsx` | напитки (раскрывающийся блок) и тарифы, добавление/редактирование |
| Статистика | `src/screens/StatsScreen.tsx` | период День/Неделя/Месяц, напитки, игры, выручка, разбивка по столам |
| Телевизор | `src/screens/TvScreen.tsx` | два стола, часы, IP для прямого Wi-Fi, тонкая вертикальная линия |

## Логика

- Два стола; каждый — свободен или открыт.
- При открытии выбирается тариф, запускается таймер.
- Сумма = время по тарифу + напитки (`src/utils/session.ts`).
- «Добавить время» добавляет минуты к сессии, «Редактировать» меняет тариф/время/напитки.
- При закрытии стола игра пишется в историю (`history`) — её читает статистика.
- Пульт отправляет на табло только снимок текущих столов, тарифов и напитков; история остаётся на пульте.

## Архитектура

```
src/
  theme.ts            дизайн-токены (цвета, отступы, тени)
  types.ts            модели данных
  store/useStore.ts   Zustand-стор: состояние + вся локальная логика + persist
  hooks/useNow.ts     тикающее «сейчас» для таймеров
  lib/sync.ts         общий слой синхронизации (cloud/LAN)
  lib/lanTransport.ts прямой TCP-транспорт для телефона и ТВ
  utils/              format (деньги/время/склонения) + session (расчёт суммы)
  components/         ui-примитивы, Page, модалки
  screens/            четыре экрана
server/
  src/index.js        socket.io + Postgres cloud-sync сервер
```
